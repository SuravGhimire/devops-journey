Topic:
Amazon S3 (Simple Storage Service)

Amazon S3 is AWS's object storage service.

It is used to store:
- Images
- Videos
- Documents
- Backups
- Application files
- Static websites

S3 is highly durable, scalable, and available from anywhere over the internet.

--------------------------------------------------------------------------------

Bucket:
------
A bucket is like a folder that stores files.

Examples:
company-backup
project-images
my-static-website

Bucket names must be globally unique.


Object:
---------
An object is the actual file stored inside a bucket.

Example:

Bucket:
my-images

Objects:
cat.jpg
resume.pdf
logo.png

Everything stored inside S3 is called an object.

-----------------------------------------------------------------
Storage Classes:

1. Standard
- Frequently accessed data

2. Intelligent-Tiering
- Automatically moves data between tiers

3. Standard-IA
- Infrequently accessed data

4. Glacier Instant Retrieval
- Archive with quick access

5. Glacier Flexible Retrieval
- Long-term archive

6. Glacier Deep Archive
- Cheapest storage
- Retrieval may take hours

Choose the storage class based on how often the data is accessed.

--------------------------------------------------------------

Common Uses:

- Website image storage
- Static website hosting
- Backup storage
- Log storage
- Software downloads
- Media files
- Data archiving

-------------------------------------

S3 Security

Access is controlled using:

- IAM Policies
- Bucket Policies
- ACL (Access Control Lists)

By default, S3 buckets are private.

Public access should only be enabled when required.

----------------------------------------------------------

Lifecycle Rules

Lifecycle rules automatically move or delete objects.

Example:

30 days
↓
Move to Standard-IA

90 days
↓
Move to Glacier

365 days
↓
Delete

This helps reduce storage costs.
