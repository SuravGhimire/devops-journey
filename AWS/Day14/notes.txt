AWS Day 14

Topic:
IAM Best Practices

1. Do not use the Root User for daily work.

2. Create IAM Users for each person.

3. Grant only the required permissions.

4. Use IAM Groups to manage permissions.

5. Rotate credentials regularly.

6. Enable Multi-Factor Authentication (MFA).


----------------------------------------------------

Principle of Least Privilege:
A user should receive only the permissions required to perform their job.

Example:

Developer

✓ Launch EC2
✓ View Logs

✗ Delete VPC
✗ Delete IAM Users

-This improves security.

--------------------------------------------------

Multi-Factor Authentication (MFA):
-MFA adds an extra layer of security.

Instead of only:

-Password

Users must also provide:

- Authenticator App Code
or
- Hardware Security Key

Benefits:

- Better account protection
- Reduces unauthorized access

AWS Well-Architected Framework

The AWS Well-Architected Framework provides best practices for designing secure, reliable, efficient and cost-effective systems.

-----------------------------

Six Pillars:

1. Operational Excellence
Improve and automate operations.

--------------------------------

2. Security
Protect systems and data.

--------------------------------

3. Reliability
Recover quickly from failures.

--------------------------------

4. Performance Efficiency
Use resources efficiently.

--------------------------------

5. Cost Optimization
Avoid unnecessary expenses.

--------------------------------

6. Sustainability
Reduce environmental impact
through efficient resource usage.

Key Takeaways:
- Never use the Root account daily.
- Enable MFA.
- Follow the Principle of Least Privilege.
- Design systems using AWS Well-Architected principles.
- Security and reliability should always be considered from the beginning.
