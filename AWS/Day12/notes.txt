AWS Day 12

Topic:
Amazon CloudWatch :

-Amazon CloudWatch is AWS's monitoring and observability service.
-It collects information about AWS resources and applications.

CloudWatch helps us:
- Monitor resources
- View performance
- Collect logs
- Create alarms
- Troubleshoot problems

* Analogy  (CloudWatch):
 
- A car's dashboard shows your speed, fuel level, and engine temperature in real time. 
CloudWatch does the exact same thing for your cloud apps, tracking metrics like CPU usage, memory, and network traffic.

---------------------------------------------------------------------

Metrics :

A metric is a measurable value.

Examples:

-EC2 CPU Utilization
-Memory Usage
-Disk Read/Write
-Network In
-Network Out

* Metrics help determine whether a resource is healthy.

--------------------------------------------

CloudWatch Logs:

Applications generate logs.

Examples:
-Application Logs
-System Logs
-Web Server Logs

CloudWatch stores these logs and makes them searchable.

Benefits:
- Debugging
- Error tracking
- Security auditing


----------------------------------------

CloudWatch Alarms:

CloudWatch continuously checks metrics.

Example:

CPU > 80%
↓
Alarm Triggered
↓
Notification Sent


Possible actions:
- Send Email
- Trigger Auto Scaling
- Invoke Lambda


---------------------------------

CloudWatch Dashboards:

A dashboard provides a visual view of AWS resources.

We can monitor:

- EC2
- RDS
- Load Balancer
- S3
- Lambda

Everything is displayed in one place.


-------------------------------------------------------------

Why DevOps Engineers Use CloudWatch ?
(Why CloudWatch?)

- Detect server problems
- Monitor application health
- Analyze logs
- Receive alerts
- Improve reliability


--------------------------

(Real-world Example)

Example:

User visits website
↓
EC2 CPU becomes 90%
↓
CloudWatch detects high CPU
↓
Alarm triggers
↓

↓
