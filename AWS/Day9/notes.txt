AWS Day 9


Topic:
Security Groups & Network ACLs


Security Group:
---------------
-A Security Group acts as a virtual firewall for an EC2 instance.

It controls:
- Incoming traffic (Inbound)
- Outgoing traffic (Outbound)

-Every EC2 instance must have at least one Security Group.

------------------------------------------------------------------

Inbound Rules

Control traffic coming INTO the EC2 instance.

Examples:

SSH
Port 22

HTTP
Port 80

HTTPS
Port 443

--------------------------------

Outbound Rules

Control traffic leaving the EC2 instance.

Example:

EC2
↓
Internet

Most AWS Security Groups allow all outbound traffic by default.

-----------------------------------------------------------------

Common Ports

22   SSH

80   HTTP

443  HTTPS

3306 MySQL

5432 PostgreSQL

8080 Web Applications
---------------------

Example

Web Server

Inbound

HTTP 80
Source:
0.0.0.0/0

HTTPS 443
Source:
0.0.0.0/0

SSH 22
Source:
My Public IP

This allows everyone to access the website,
but only I can SSH into the server.


------------------------------------------------

Network ACL (Access Control List)

A Network ACL protects an entire subnet.

Unlike Security Groups,
Network ACLs work at the subnet level.

Security Group
↓
Protects an EC2 Instance

Network ACL
↓
Protects the entire Subnet


------------------------------

Security Group:

- Instance Level
- Stateful
- Only Allow Rules

Network ACL:

- Subnet Level
- Stateless
- Allow and Deny Rules

-----------------------

Stateful:

If inbound traffic is allowed,
the response is automatically allowed.

Example:

Laptop
↓
SSH
↓
EC2
↓
Reply automatically allowed.


Stateless:

Both inbound and outbound
rules must be configured separately.

------------------------------------------

Internet
      │
      ↓
Security Group
      │
      ↓
EC2 Web Server
      │
-----------------------
Subnet
Protected by
Network ACL
-----------------------



--------------------------------------------

Best Practices

- Allow only required ports.
- Never allow SSH from everyone.
- Use your public IP for SSH access.
- Keep Security Groups simple.
- Use Network ACLs for extra protection.
