AWS Day 20

Topic:
AWS CLI Configuration & Profiles

Objective:
Learn how AWS CLI stores credentials and how to manage multiple profiles.

------------------------------------------------------------

Important Observation

When using AWS CLI without specifying AWS_ENDPOINT_URL, the CLI connects to the real AWS cloud.

Since I used dummy credentials (devuser/devpassword), AWS returned an "InvalidAccessKeyId" error.

When I specified:

AWS_ENDPOINT_URL=http://localhost.floci.io:4566

the AWS CLI connected to my local Floci server instead of the real AWS cloud, and the command executed successfully.

This shows how the AWS CLI can communicate with different endpoints while using the same commands.

----------------------------------------------------

What is AWS CLI Configuration?

AWS CLI stores configuration in two files:

~/.aws/credentials
Stores:
- Access Key ID
- Secret Access Key

~/.aws/config
Stores:
- Region
- Output format
- Other CLI settings

------------------------------------------------------------

What is a Profile?

A profile is a named set of AWS credentials and configuration.

Examples:
- default
- dev
- production

Profiles allow you to work with multiple AWS accounts without repeatedly changing credentials.

------------------------------------------------------------

Commands Practiced

aws configure
Configure the default profile.

aws configure --profile dev
Create a new profile.

aws configure list
Show current configuration.

aws configure list-profiles
Display all available profiles.

aws s3 ls --profile dev
Run AWS CLI commands using a specific profile.

------------------------------------------------------------

Real World Uses

- Working with multiple AWS accounts
- Development and production separation
- Team environments
- CI/CD pipelines

------------------------------------------------------------

Summary

Today I learned how AWS CLI stores credentials and configuration. I created and used multiple CLI profiles, making it easier to switch between environments without changing credentials manually.
