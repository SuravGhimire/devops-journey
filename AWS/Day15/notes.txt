Other AWS Services:

- IAM manages user permissions.
- Security Groups protect resources.
- CloudWatch monitors the application.
- Auto Scaling adds/removes EC2 instances.
- Amazon S3 stores files and backups.


---------------------------------------------------

{Request Flow}

How a User Request Travels:

1. User enters:

www.myapp.com
↓
2. Route 53 resolves the domain name.
↓
3. Traffic reaches the Load Balancer.
↓
4. Load Balancer forwards traffic to a healthy EC2 instance.
↓
5. EC2 processes the request.
↓
6. If data is needed,EC2 communicates with Amazon RDS.
↓
7. Response is sent back to the user.

----------------------------------------------------------------------

Best Practices:

- Never use the Root account for daily work.
- Enable MFA.
- Follow the Principle of Least Privilege.
- Use Security Groups to restrict access.
- Keep databases private.
- Enable CloudWatch monitoring.
- Use Auto Scaling for high traffic.
- Store static files in S3.
- Deploy across multiple Availability Zones.
