AWS Day 2

Topic:
IAM (Identity and Access Management)

IAM is used to control:

- Who can access AWS
- What they can access
- What actions they can perform

IAM Components:

1. User
2. Group
3. Policy

Purpose:
Security and access control in AWS.

-----------------------------------------------------------------------------


IAM User:
Individual AWS account identity.

IAM Group:
Collection of users with similar permissions.

IAM Policy:
JSON document defining permissions.

Best Practice:
Assign permissions through groups.
