Amazon RDS (Relational Database Service):

-Amazon RDS is a managed database service provided by AWS.
-Instead of installing and managing a database server yourself, AWS manages it for you.

Benefits:
- Easy setup
- Automatic backups
- Software updates
- High availability
- Monitoring

--------------------------------------------

Why RDS?

Without RDS:
------------
Install Linux
↓
Install MySQL
↓
Configure database
↓
Manage backups
↓
Apply security updates

--------------------------------

With RDS:
---------
Create RDS Instance
↓
AWS manages everything


---------------------------------------------

Amazon RDS supports:

- MySQL
- PostgreSQL
- MariaDB
- Oracle
- Microsoft SQL Server

Choose the engine based on your application requirements.

--------------------------------------------------------------

(Multi-AZ Deployment):

Multi-AZ:
---------
Primary Database
        │
Automatic Replication
        │
Standby Database

If the primary database fails, AWS automatically switches to the standby database.

Benefit:
High Availability


----------------------------------------------

Read Replica

Primary Database
      │
Replication
      │
Read Replica

Purpose:
Handle read-heavy workloads.

Applications can read data from replicas , while writes continue to the primary database.

-----------------------------------

Automatic Backups:
- Enabled by AWS
- Used for point-in-time recovery

Snapshots:
- Manual backups
- Created by the user
- Can be restored later

--------------------------


Best Practices

- Enable automatic backups
- Use Multi-AZ for production
- Use Read Replicas for scaling reads
- Restrict database access using Security Groups
- Never expose a production database directly to the internet
