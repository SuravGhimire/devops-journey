AWS Day 13

Topic:
Amazon Route 53:

-Amazon Route 53 is AWS's Domain Name System (DNS) service.
-It translates domain names into IP addresses.

Example:

www.example.com
↓
192.168.1.10

Instead of remembering IP addresses, users access websites using domain names.

----------------------------------------------------


(How DNS Works)
DNS Resolution

User enters:
-----------
www.example.com
↓
DNS Query
↓
Route 53
↓
Returns IP Address
↓
Browser connects to server

-----------------------------------------------------------

Hosted Zone

A Hosted Zone stores DNS records for a domain.

Example:
example.com

Records:
-A Record
-AAAA Record
-CNAME
-MX
-NS


----------------------------------------------

DNS Record Types:

A Record:
Maps a domain to an IPv4 address.

AAAA Record:
Maps a domain to an IPv6 address.

CNAME:
Creates an alias to another domain.

MX Record:
Specifies the mail server for a domain.

NS Record:
Specifies the authoritative name servers.


-----------------------------------------------------

Routing Policies:
----------------
Simple Routing:
One resource.

Weighted Routing:
Traffic distributed based on percentages.

Latency Routing:
Routes users to the region with the lowest latency.

Failover Routing:
Routes traffic to a backup resource if the primary fails.

Geolocation Routing:
Routes users based on geographic location.


--------------------------------------------------------------

(Why Route 53)
Benefits

- Highly available
- Scalable
- Fast DNS resolution
- Health checks
- Supports multiple routing policies
